# Simple Restart Plan (No Big Dump, Small Daily Steps)

## Goal
Keep project easy to explain in Zoom and match teacher requirements with small commits per day.

## Day 1 (today) - Explanation first
- Add simple essay (`docs/simple-seminar-essay.md`)
- Add simple day-by-day plan (this file)
- Do not change core logic heavily

## Day 2
- Choose one small live-change target (example: add one new `visitType`)
- Update backend constants + frontend options in one place
- Run lint/build and commit

## Day 3
- Add one small UI improvement teacher can ask for live
- Example: add one new filter field or one table column
- Verify and commit

## Day 4
- Add one small backend improvement
- Example: clearer validation message or extra guard
- Verify and commit

## Day 5
- Prepare final report notes:
  - architecture
  - data model
  - endpoint list
  - validations
- Commit docs only

## Day 6
- Seminar rehearsal pass:
  - run app from scratch
  - do one live change
  - explain each layer
- Commit tiny cleanup if needed

## Push Strategy
- Push one small commit at a time.
- Keep commit messages short and specific.
- Do not push multiple days at once.
